---

## מה תקבל כל שבוע?

**התראת Push בפלאפון** (מגיעה כ-40 דקות אחרי 8:00 בבוקר, כשהכל מוכן) עם שלושה כפתורים:

---

## מה עומד מאחורי הקלעים?

שני דברים שחשוב להבחין ביניהם:

- **NotebookLM** — שירות הענן של Google ב-notebooklm.google.com. לא מותקן שום דבר — אתה פותח אותו בדפדפן כרגיל.
- **notebooklm-py** — תוכנה קטנה (לא רשמית) של מפתח עצמאי, שמתקינים פעם אחת על המחשב. היא יודעת לתקשר עם שירות Google NotebookLM דרך ממשקים פנימיים, בדיוק כמו שהדפדפן שלך מתקשר איתו — רק בצורה אוטומטית.

כשמריצים `notebooklm login`, התוכנה פותחת דפדפן, אתה מתחבר עם חשבון Google שלך, והיא שומרת את ה-cookies (עוגיות ההתחברות) בקובץ מקומי. אותן cookies בדיוק מועברות לאחר מכן ל-GitHub, ומאפשרות ל-GitHub Actions להתחבר ל-NotebookLM מהענן — בלי שהמחשב שלך יהיה דלוק.

## שלב 3 — התקן את notebooklm-py וקבל את ה-cookies

**זה מה שמאפשר ל-GitHub להתחבר ל-NotebookLM שלך.**

פתח Terminal ורשום בזה אחר זה:

```bash
pip install notebooklm-py
playwright install chromium
notebooklm login
```

הפקודה האחרונה תפתח דפדפן — התחבר עם חשבון Google שלך (אותו חשבון שאתה משתמש בו ב-NotebookLM).

לאחר ההתחברות, חזור ל-Terminal והרץ:

```bash
cat ~/.notebooklm/storage_state.json
```

יופיע טקסט ארוך שמתחיל בסימן `{`. **העתק אותו כולו** — תצטרך אותו בשלב הבא.

> **חשוב:** cookies אלה יפוגו עם הזמן (בדרך כלל אחרי כמה חודשים).
> כשהמערכת מפסיקה ליצור מחברות, חזור על שלב זה ועדכן את ה-Secret בגיטהאב.

## שלב 5 — הוסף את הסודות ל-GitHub

1. ב-repository שיצרת → לחץ **Settings**
2. בתפריט השמאלי: **Secrets and variables → Actions**
3. לחץ **New repository secret** והוסף שניים:

| Name | Value |
|------|-------|
| `NOTEBOOKLM_AUTH_JSON` | הטקסט שהעתקת בשלב 3 |
| `NTFY_TOPIC` | שם הערוץ שבחרת בשלב 4 |

## פתרון בעיות נפוצות

**לא נוצרה מחברת / שגיאת Authentication:**
→ ה-cookies פגו תוקף. חזור על שלב 3 ועדכן את `NOTEBOOKLM_AUTH_JSON` ב-Settings.

**לא קיבלתי התראה:**
→ ודא שה-`NTFY_TOPIC` ב-GitHub זהה לשם הערוץ באפליקציה.

**הריצה נכשלה:**
→ לחץ על הריצה ב-Actions → פתח את ה-logs לפרטים.

**לא נמצאו מאמרים:**
→ PubMed אולי עמוס. הרצה ידנית שוב תפתור זאת.
